Earliest Deadline First

Earliest Deadline First is dynamic preemptive scheduling algorithm, in which the priority of task us assigned according to absolute deadline. This deadline is also compared every time the new task is added.

  • Advantages
      • This is optimal scheduling, means if a task missed deadline the processor is 100% in use.
      Best Suited for periodic Tasks
      EDF can be used upto 100% utilization bound
      It ensure no deadline miss according to the utilization bound.
  • Disadvantages
      • Not all real time tasks are periodic and if the period is not known these policies can not be used.
      In case the utilization upper bound of EDF is not satisfied the tasks are not schedulable.
